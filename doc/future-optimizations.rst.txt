Overview
========


Significant optimizations have been applied to the seed7-mode code to speed up
the indentation control.  The next step is now to ensure that all indentation
and navigation is done correctly for all existing Seed7 code.

This file stores pending ideas for further optimizations that could be applied
later once the focus on indentation and navigation validity along with
automatic testing is complete.



To Speed up Indentation control done by seed7-calc-indent
=========================================================


seed7--current-line-nth-word is convenient but relatively heavy for hot indentation checks
------------------------------------------------------------------------------------------

``seed7-calc-indent`` calls ``seed7--current-line-nth-word`` at Line 5754 for every
non-trivial indentation calculation, and again at Lines 5909-5910 for end
block.

``seed7--current-line-nth-word`` uses ``forward-word`` and ``thing-at-point``,
which are syntax-table-aware and more general than needed for a few keyword
checks.

Potential future optimization: add a specialized helper for the first/second
simple word on the current line, implemented with skip-chars-forward and
buffer-substring-no-properties or direct looking-at checks.

For example, for the first word:

.. code:: elisp

          (defun seed7--current-line-first-word-fast ()
            "Return first simple word on the current line, or nil."
            (save-excursion
              (forward-line 0)
              (skip-chars-forward " \t")
              (when (looking-at "\\([[:alnum:]_$]+\\)")
                (match-string-no-properties 1))))

Or, even faster for the current seed7-calc-indent cases, avoid extracting
words for special cases and use direct line-prefix checks only where needed.


The top-level block-start “noop insertion” path is correct-looking but expensive
--------------------------------------------------------------------------------

In seed7-line-inside-a-block, Lines 4752-4767 temporarily insert " noop;\n"
and recursively call seed7-calc-indent.

This is guarded and probably not frequent in the large-file end-of-file case,
but it is one of the most expensive operations in the indentation subsystem
because it temporarily mutates the buffer, updates markers, and re-enters the
indentation engine.

If performance profiling still shows hotspots, this is a candidate for a
non-mutating simulation. For example, a dedicated helper could compute “indent
as if previous line were a noop” using the surrounding block spec rather than
physically inserting a line.
